Noah Memory – Privacy Policy

תמצית

Noah Memory API שומר ומעבד מידע אך ורק לפי בקשת המשתמש ולצרכי תפקוד ה־Custom GPT Noah – Presence & Insight. איננו מוכרים מידע ואיננו משתפים צד ג' ללא הוראה מפורשת של המשתמש.

איזה מידע נשמר

  • תכנים שהמשתמש מבקש לשמור במפורש (למשל: הערות, פרטים תפעוליים, פרופיל עבודה).
  • מטא־נתונים טכניים לצורך אבטחה וניטור תקין של השירות (למשל: חותמת זמן, מזהה בקשה).

אין איסוף של מידע רגיש (בריאות, דת, פוליטיקה וכו') אלא אם המשתמש מבקש ושומר זאת במפורש.

מטרות השימוש

  • שליפה הקשרית של זיכרון לשיפור החוויה והשיחה.
  • שימור העדפות עבודה (למשל: סגנון כתיבה/שפה).
  • אבטחה ותפעול השירות.

שיתוף מידע

אין שיתוף מידע עם צד ג' אלא אם המשתמש מבקש זאת במפורש או אם הדבר נדרש לפי חוק.

משך שמירה ומחיקה

  • ברירת מחדל: שמירה עד בקשת מחיקה.
  • מחיקה: ניתן לבקש ממחיקת זיכרון נקודתית ועד מחיקה מלאה דרך הפקודות/הממשק.

אבטחה

  • גישה מאובטחת ל־API באמצעות מפתח גישה (API Key).
  • הפרדת סביבות פיתוח/בדיקות/ייצור כאשר רלוונטי.
  • לוגים מצומצמים וללא תוכן אישי, לצורך תפעול בלבד.

זכויות משתמש

  • עיון במידע שנשמר לגביך (באמצעות listMemory).
  • הוספה/עדכון/מחיקה של פריטים (באמצעות addMemory ופקודות נלוות).
  • פנייה אלינו בכל שאלה/בקשה.

יצירת קשר

לשאלות ובקשות: contact@example.com. ניתן לעדכן לכתובת מועדפת.

Summary

Noah Memory API stores and processes information strictly at the user's request and solely to operate the Custom GPT Noah – Presence & Insight. We do not sell data and do not share it with third parties without explicit user direction.

Data We Store

  • Content the user explicitly asks to store (e.g., notes, operational details, work profile).
  • Limited technical metadata for security and service integrity (e.g., timestamp, request id).

We do not collect sensitive categories (health, religion, politics, etc.) unless the user explicitly asks to store such content.

Purpose of Use

  • Contextual memory retrieval to improve the experience and dialogue.
  • Persisting work preferences (e.g., style/language).
  • Service security and operations.

Sharing

No data is shared with third parties unless explicitly requested by the user or required by law.

Retention & Deletion

  • Default: retained until deletion is requested.
  • Deletion: from single-item erase up to full wipe via commands/UI.

Security

  • API access via API Key.
  • Environment separation (dev/test/prod) where applicable.
  • Minimal logs without personal content, for ops only.

User Rights

  • View your stored data (via listMemory).
  • Add/Update/Delete items (via addMemory and related endpoints).
  • Contact us with any request.
